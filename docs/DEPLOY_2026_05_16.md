## 2. 管理者 UID に Custom Claim を付与

**これをやらないと `adminCancelTransaction` が全員 403 になる。**

> 補足：`fixSellerStatus` 関数は 2026-05-16 にセキュリティ修正で削除済み（モック Stripe Connect ID を生成して KYC を擬似的に通せる構造だったため）。本書旧版で参照していたが、現在は存在しない。

### 2-1. サービスアカウントキーを準備

1. Firebase Console → プロジェクト設定 → サービスアカウント
2. 「新しい秘密鍵の生成」をクリック
3. ダウンロードしたJSONを `functions/serviceAccountKey.json` として保存
   （`*-key.json` は `.gitignore` 済みなのでコミットされない）

### 2-2. スクリプト実行

```bash
# メールアドレス指定
node scripts/grant-admin-claim.js admin@musashino-u.ac.jp

# または UID 直接指定（Firebase Console → Authentication で確認可能）
node scripts/grant-admin-claim.js --uid abc123xyz

# 取り消す場合
node scripts/grant-admin-claim.js admin@musashino-u.ac.jp --revoke
```

実行後、対象ユーザーは**一度サインアウト→再サインイン**が必要（ID トークンを更新するため）。

## 3. Firestore Rules を本番にデプロイ

```bash
firebase use production   # 本番エイリアスに切り替え (.firebaserc 参照)
```

デプロイ後、以下を **Firebase Console → Firestore → ルール** で目視確認：
- `conversations` / `messages` の rules が `allow read, write: if false;` になっている
- `users/{userId}` に `userServerOnlyFields()` のチェックが入っている
- `transactions/{id}` の create rule から `anonymous` 関連の bypass が消えている

## 4. Cloud Functions を本番にデプロイ

> **重要**：デプロイ前に古いビルド成果物を必ず削除してから再ビルドする。
> `functions/lib/` にはセッション前に削除した `onMessageCreated` / `fixSellerStatus` の `.js` が残っている可能性があり、誤ってデプロイされると削除済みの関数が復活する。

```bash
# 1. クリーンビルド（削除済み関数の痕跡を排除）
rm -rf functions/lib functions/tsconfig.tsbuildinfo
pnpm --prefix functions run build

# 2. 本番デプロイ
firebase use production
firebase deploy --only functions
```

デプロイ後、Firebase Console → Functions で以下を確認：
- `onMessageCreated` が **存在しない**（チャット削除で除去）
- `fixSellerStatus` も **存在しない**（KYC バイパスリスクで除去）
- `adminCancelTransaction` / `capturePayment` / `unlockTransaction` / `createPaymentIntent` / `stripeWebhook` / `cancelTransaction` / `rateUser` / `cancelStaleTransactions` 等の必要な関数がデプロイされている
- 直近のログにエラーが出ていない

## 7. ロールバック手順（万一の事故時）

```bash
# 1. ひとつ前のリリースコミットに戻す
git checkout main
git revert <NEW_COMMIT_SHA> --no-edit
git push

# 2. 再デプロイ
firebase use production
firebase deploy --only firestore:rules,functions,hosting
```

ただし **Custom Claim の付与は revert で消えない**（Auth は別管理）。
必要なら `scripts/grant-admin-claim.js --revoke` で個別に取り消す。
