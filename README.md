## 📌 プロジェクトの背景・目的 (Project Background)

本プロジェクトは、**経済学部生による「学内経済圏の課題解決」を目的とした実証実験**です。

### 学内市場における「情報の非対称性」と「資源の死蔵」
現在、大学内では毎年大量の教科書が、使い終わった後に捨てられるか、二束三文で外部市場（ブックオフ等）に流出しています。一方で、新入生は高額な定価で教科書を購入しており、学全体で見ると**莫大な経済的厚生の損失 (Deadweight Loss)** が発生しています。

「Musashino Link」は、この需給の不一致をWebテクノロジーによって解消し、**「学内の教科書は、学内で循環させる (Circular Economy)」** という新しい経済合理性を確立するためのプラットフォームです。

> **Note**: 本プロジェクトは学生個人の発案による実験的な取り組みであり、大学の公式システムではありません。

## 🌏 社会的意義とSDGs (Social Impact)

本システムは、単なるフリマアプリではなく、以下の社会的価値の創出を目指しています。

### 1. 資源利用の最適化 (SDGs 12: つくる責任、つかう責任)
一度使われただけの良質な教科書を廃棄せず、次の学習者へバトンタッチすることで、紙資源の無駄を削減します。

### 2. 学生の経済的負担の軽減 (Economic Efficiency)
外部の中古書店やフリマアプリで発生する「中間マージン」や「物流コスト（送料・梱包費）」を、**学内手渡し**によってゼロにします。
浮いたコストは全て学生（売り手・買い手）に還元され、実質的な可処分所得の向上に寄与します。

### 3. 安全な取引市場の設計 (Market Design)
既存のSNS（Xなど）での個人間取引は、詐欺やトラブルのリスクが高い「不完全な市場」です。
本システムでは、**エスクロー決済（仮払い）** と **学籍番号認証** を導入することで、信頼コストを担保し、誰もが安心して参加できる健全な市場環境を提供します。

## 💡 機能的特徴 (Features)

### 1. 学内限定の認証ネットワーク
武蔵野大生しかログインできない認証基盤を構築。外部の不審者を完全に排除し、コミュニティ内の信頼を担保します。

### 2. エスクロー決済 (Stripe Connect)
「お金を払ったのに商品が来ない」「商品を渡したのにお金が貰えない」という**カウンターパーティ・リスク**をシステム構造的に排除しました。

### 3. "ゼロヒット" 需要分析
検索ログを解析し、「今、どの学部のどの教科書が足りていないか」という**需要の可視化**を行います。これにより、埋もれている在庫（先輩の本棚）を効率的に市場へ呼び込むことが可能です。

## 🛠️ 技術スタック (Technology Stack)
経済学的課題を解決するための手段として、以下のモダンなWeb技術を採用しています。

| カテゴリ | 技術 | 選定理由 |
| :--- | :--- | :--- |
| **Frontend** | **Next.js 14** | ストレスのない高速な取引体験の提供 |
| **Backend** | **Firebase** | リアルタイムな需給マッチングと認証基盤 |
| **Payment** | **Stripe Connect** | C2C取引における資金保全（エスクロー） |

---

## 🛡️ セキュリティとプライバシー
*   **データ保護**: 通信は全てSSL/TLSで暗号化。
*   **権限管理**: Firestoreセキュリティルールによる厳格なアクセス制御。
*   **決済情報**: PCI DSS準拠のStripeが管理し、サーバー内には一切保持しません。
